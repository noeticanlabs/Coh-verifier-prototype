CohBit = A cryptographically committed, certificate-verified, budget-bounded state transition.

C7: rv_ok
Certificate verification status must be accept. This is the gatekeeper — no execution without proper cryptographic commitment.
C6: cert_ok
The certificate must certify the specific state transition: from_state → action → to_state
C17: exec_ok
The execution function must compute the correct target state from the source state and action.
C8: defect_ok
Defect (coherence loss) must be bounded above by delta_hat (maximum tolerable defect).
C9: margin_ok
Budget admissibility: V(to_state) + spend ≤ V(from_state) + defect + authority
Theorem: Identity CohBit Exists
For any state x ∈ X and certificate cert such that S.rv_verify cert = accept and S.certificates cert x id_action x, there exists a valid CohBit: CohBit S = { from_state := x, action := id_action, to_state := x, cert := cert } This grounds the transition graph in a formally admissible neutral element — the identity transition leaves the state unchanged.
Theorem: Chain Stability (Telescoping Sum)
For a verified chain of CohBits (bit₀, bit₁, ..., bitₙ) where continuity holds: V_n + sum(spend) <= V_0 + sum(defect) + sum(authority) The cumulative coherence budget is preserved across the entire execution trace. This is the crown jewel — the budget constraint telescopes across the chain.
